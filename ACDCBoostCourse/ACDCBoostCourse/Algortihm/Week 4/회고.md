# 알고리즘 회고

변수를 선언할 때 a, b, c로 선언하는 것이 아니라 조금 더 목적에 맞게 alphabetCount 등으로 선언하는 것이 코드 읽을 때 더 용이하다는 것을 알게 됨.

또한, 다음 알고리즘 문제를 해결할 때는 UpperCamelCase, lowerCamelCase을 적용해봐야겠다.

알고리즘 문제를 해결할 때, 한 가지 방법을 생각해내고 다른 방법이 있는지 한번 더 생각해보는 것도 좋을 것 같다.

알고리즘 문제를 해결하기 위해 추가적으로 적어주는 헤더파일들 구글링으로 무작정 검색하지 말고 조금씩이라도 암기할 필요성이 있어 보인다..
